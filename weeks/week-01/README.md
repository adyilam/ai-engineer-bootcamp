# 👋 Welcome

Welcome to the **AI Engineer Bootcamp**.

Artificial Intelligence is transforming how software is designed and built. Modern applications are no longer just CRUD systems—they can understand language, generate content, search knowledge, reason about problems, and assist users in ways that were impossible only a few years ago.

The role of the software engineer is evolving, and organizations are looking for engineers who can combine **software engineering principles** with **AI capabilities**.

This bootcamp is designed to help you become one of those engineers.

---

# 🤖 What Is AI Engineering?

AI Engineering is the discipline of designing, building, deploying, and maintaining AI-powered software applications.

Unlike Machine Learning Engineers who primarily build and train models, AI Engineers focus on integrating AI into real-world applications.

An AI Engineer combines knowledge from several disciplines:

- Software Engineering
- Artificial Intelligence
- Cloud Computing
- APIs
- Distributed Systems
- Databases
- DevOps
- Security

The goal is not to build a language model from scratch—it is to build reliable, scalable applications that use AI effectively.

---

# 🧩 AI Engineer vs Other Roles

| Role | Primary Focus |
|------|----------------|
| Software Engineer | Builds business applications and backend systems |
| Data Scientist | Analyzes data and builds predictive models |
| Machine Learning Engineer | Trains and deploys machine learning models |
| AI Engineer | Builds AI-powered applications using existing models and AI frameworks |

In this bootcamp, our focus is **AI Engineering**.

# 🛠 Engineering Standards

Throughout this bootcamp, we will follow professional software engineering practices.

## Version Control

- Use Git for source control.
- Write meaningful commit messages.
- Work with feature branches when appropriate.

## Documentation

- Every project will include a README.
- Every lesson will have clear objectives and summaries.
- Architectural decisions will be documented.

## Code Quality

- Write clean, readable code.
- Keep functions and classes focused.
- Favor simplicity over complexity.

## Testing

- Test what you build.
- Validate inputs.
- Handle errors gracefully.
